id Long AUTO_INCREMENT PK
content String NOT NULL
completed boolean DEFAULT false
dueDate LocalDate NULLABLE
priority Priority DEFAULT MEDIUM
createdAt LocalDateTime AUTO TIMESTAMP
Priority Enum
HIGH MEDIUM LOW

DB에는 VARCHAR(10)으로 저장. Enum.valueOf()로 변환.

DDL Schema
CREATE TABLE todo ( id BIGINT AUTO_INCREMENT PRIMARY KEY, content VARCHAR(255) NOT NULL, completed BOOLEAN DEFAULT FALSE, due_date DATE, priority VARCHAR(10) NOT NULL DEFAULT 'MEDIUM', created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP );
설계 포인트

— ORM 미사용으로 SQL 동작을 명시적으로 파악 가능

— created_at DB DEFAULT 설정으로 앱 의존도 제거

— due_date NULL 허용으로 선택적 마감일 지원

— Priority는 VARCHAR 저장 후 Enum 변환 처리

PUT /api/todos/{id} Path: id, Body: Todo Todo 특정 할 일 수정
DELETE /api/todos/{id} Path: id 204 No Content 단건 삭제
DELETE /api/todos/completed — deletedCount 완료 항목 일괄 삭제
GET /api/todos 쿼리 파라미터
status ALL · ACTIVE · COMPLETED (기본값: ALL)
query content 키워드 검색. 대소문자 무관 LIKE 처리
sort LATEST (최신순) · DUE_ASC (마감일순, NULL은 후순위) (기본값: LATEST)

Core Code
Repository 핵심 구현
동적 SQL 빌더 — findAll()
StringBuilder sql = new StringBuilder( "SELECT * FROM todo WHERE 1=1"); // 상태 필터 if ("ACTIVE".equals(status)) sql.append(" AND completed = FALSE"); else if ("COMPLETED".equals(status)) sql.append(" AND completed = TRUE"); // 키워드 검색 if (query != null && !query.isBlank()) sql.append(" AND LOWER(content) LIKE ?"); // 정렬 if ("DUE_ASC".equals(sort)) sql.append(" ORDER BY due_date ASC NULLS LAST"); else sql.append(" ORDER BY created_at DESC");
구현 포인트

— WHERE 1=1 패턴으로 조건을 동적으로 추가

— LOWER()로 대소문자 무관 검색 구현

— NULLS LAST로 마감일 없는 항목 후순위 정렬

RowMapper 구현
private final RowMapper<Todo> todoRowMapper = (rs, rowNum) -> { Long id = rs.getLong("id"); String content = rs.getString("content"); boolean completed = rs.getBoolean("completed"); LocalDate due = rs.getObject("due_date", LocalDate.class); Priority p = Priority.valueOf(rs.getString("priority")); LocalDateTime ca = rs.getObject("created_at", LocalDateTime.class); return new Todo(id, content, completed, due, p, ca); };

검색 · 필터 · 정렬

우선순위 관리
HIGH / MEDIUM / LOW 3단계로 업무 중요도를 분류. DB는 VARCHAR 저장, Java는 Enum으로 처리
완료 항목 일괄 삭제
완료된 할 일을 한 번에 삭제하는 벌크 DELETE API. 삭제된 건수를 응답으로 반환
마감일 추적
LocalDate 타입으로 선택적 마감일 설정 가능. DUE_ASC 정렬 시 마감일 없는 항목은 NULLS LAST 처리
REST + MVC 컨트롤러 분리
API 전용 TodoController와 View 전용 TodoViewController를 분리해 역할 명확화 및 유지보수성 향상

직접 SQL 제어
